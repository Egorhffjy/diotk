# ДиОТК — мобильная версия навигатора источников (PWA)

Это **мобильная/PWA** версия навигатора по вопросам 1–67.
Работает на телефоне **в браузере** и может быть установлена «как приложение» на главный экран.

## Что умеет
- Выбор вопроса по номеру (1–67) или из списка
- Список интернет-источников (кликабельно)
- Список презентаций/файлов + диапазоны слайдов (если есть)
- Если `sources` пустой — автоматически показываются поисковые ссылки (Google/Яндекс/Scholar)
- Можно **импортировать свой** `questions.json` (кнопка в настройках) — файл сохранится в памяти браузера (localStorage)

## Как запустить на ПК (для теста)
Нужно открыть папку через **любой статический сервер** (из-за загрузки JSON).

### Вариант 1: Python
1) Открой терминал в папке проекта
2) Выполни:

```bash
python -m http.server 8080
```

3) Открой в браузере: `http://localhost:8080`

### Вариант 2: Visual Studio Code (Live Server)
- Установи расширение **Live Server**
- Запусти `index.html`

## Как пользоваться на смартфоне
1) Размести папку на сервере (или запусти сервер на ПК и открой сайт с телефона по Wi‑Fi)
2) Открой сайт в мобильном браузере
3) Добавь на главный экран:
   - iOS Safari: **Поделиться → На экран «Домой»**
   - Android Chrome: **⋮ → Add to Home screen / Установить**

## Про файлы презентаций
В `questions.json` в блоке `presentations` можно указать:

```json
{
  "fileName": "Тестирование ЭУ.pptx",
  "slideFrom": 12,
  "slideTo": 18,
  "note": "JTAG / boundary-scan"
}
```

Если в настройках указать `Базовый URL для файлов`, то имя файла станет ссылкой: `baseURL + fileName`.
Например, если ты выложил презентации в `/docs/`, то поставь `/docs/`.

## Формат questions.json
Файл должен иметь вид:

```json
{
  "questions": [
    {
      "number": 1,
      "title": "...",
      "sources": [{"title": "...", "url": "https://..."}],
      "presentations": [{"fileName": "...", "slideFrom": 1, "slideTo": 3, "note": "..."}]
    }
  ]
}
```
